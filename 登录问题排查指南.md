# 登录问题排查指南

## 问题描述
无法登录，提示"账号密码错误"

## 可能的原因

### 1. 数据库问题
- 数据库文件可能损坏或丢失
- 用户数据可能被清空
- 数据库路径配置问题

### 2. 密码哈希问题
- 密码哈希算法可能不一致
- 存储的哈希值可能有问题

### 3. 用户数据丢失
- 如果之前注册的用户数据丢失，需要重新注册

## 解决方案

### 方案1：使用默认测试账号
系统提供了默认测试账号：
- **邮箱**: admin@example.com
- **密码**: admin123

或者：
- **邮箱**: test@example.com
- **密码**: test123

### 方案2：重新注册账号
1. 在登录页面选择"註冊"模式
2. 输入新的邮箱和密码（至少6个字符）
3. 注册成功后会自动登录

### 方案3：检查数据库
如果使用数据库模式：
1. 检查数据库文件是否存在
2. 检查数据库路径配置
3. 可能需要重新初始化数据库

### 方案4：使用内存模式
如果数据库有问题，可以：
1. 切换到内存模式（测试用）
2. 重新注册账号
3. 注意：内存模式的数据不会持久化

## 调试步骤

1. **尝试默认账号**
   - 使用 admin@example.com / admin123
   - 或 test@example.com / test123

2. **重新注册**
   - 如果默认账号不行，使用注册功能创建新账号

3. **检查数据库**
   - 查看侧边栏的数据库路径
   - 检查数据库文件是否存在

4. **查看错误信息**
   - 登录时会显示具体错误信息
   - 根据错误信息进行相应处理

## 预防措施

1. **定期备份数据库**
2. **记录账号信息**
3. **使用强密码**
4. **避免使用内存模式存储重要数据**

---

**如果以上方法都无法解决，请检查：**
- 数据库文件权限
- 数据库文件是否损坏
- 系统日志中的错误信息
