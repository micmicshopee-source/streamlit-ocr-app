# 側邊欄樣式與排版方案（參考 Google AI Studio 截圖）

**原則**：依參考圖的樣式與排版調整側邊欄，**功能與文案內容不變**（仍為發票報帳、AI 合約比對、AI 客服小秘、AI 會議精華、設定、下雪、用戶、登出；不展示「取得 API 金鑰」）。

---

## 一、參考圖視覺摘要

### 1.1 整體風格
- **背景**：整欄深黑（#1e1e1e），無其他色塊。
- **字體**：無襯線、乾淨，無按鈕粗框，偏「科技感」。
- **文字色**：未選中為淺灰，標題與選中項為白或略亮灰。

### 1.2 頂部品牌區
- **主標題**：大字、白色，置頂（如「Google AI Studio」）。
- **導航**：緊接標題下一行，「‹ 儀表板」、淺灰、無底線、左箭頭。

### 1.3 主導航區（選單群組化）
- **呈現**：簡約文字導航，非按鈕式。
- **未選中**：純文字、淺灰、無背景。
- **選中態**：一項有**深灰／半透明灰、圓角矩形背景**，文字略亮（淺灰或白），與其他項有明顯區隔。
- **間距**：項目之間有適當垂直間距，整體不擠。

### 1.4 底部固定區（Sticky Bottom）
- **位置**：與主導航之間有較大留白，視覺上「壓在底部」。
- **分隔**：上方有一條分隔線。
- **內容從上到下**：
  - 趣味開關（如「※ 下雪吧」+ 圖示）。
  - 設定（齒輪圖示 + 文字）。
  - （參考圖有「取得 API 金鑰」；本專案**不展示**此項。）
  - 用戶區：**圓形頭像**（橘底、白字首字如 `m`）+ **Email 截斷**（如 `micmicshopee@gma...`），橫排、淺灰字。

### 1.5 細節
- **按鈕／可點項**：無粗框、無明顯邊框，hover 可略變色。
- **圖示**：單色（白／淺灰）、簡潔，與文字對齊。
- **外鏈**：若有「更新日誌」類，可加小箭頭圖示（本專案可選做）。

## 三、樣式與排版規格（對齊參考圖）

### 3.1 色彩
- 側邊欄背景：`#1e1e1e`。
- 一般文字：`#b0b0b0`（或同級淺灰）。
- 主標題、選中項文字：`#ffffff` 或略低對比白。
- 選中項背景：`rgba(255,255,255,0.1)` 或深灰圓角塊。
- 用戶頭像：橘底（如 `#f57c00`）、白字。

### 3.2 字體與間距
- 字體：系統無襯線或微軟正黑體，與主站一致。
- 主標題：較大、字重略重、與「‹ 儀表板」間距小。
- 導航項：垂直間距約 8–12px，可依 radio 預設再微調。
- 底部區：與主導航之間留白明顯（spacer 或 flex 撐開）。

### 3.3 選中態（主導航）
- 背景：灰/半透明白、圓角（如 8px）、padding 約 8px 12px。
- 無額外邊框、無陰影，與參考圖一致。

### 3.4 按鈕與可點元素
- 側邊欄內按鈕：`border: none`、無 box-shadow、背景透明。
- Hover：背景略亮（如 `rgba(255,255,255,0.08)`）、字色可略亮。

### 3.5 底部區排版
- 分隔線後：設定（Expander）→ 下雪開關 → 用戶頭像 + Email（橫排）→ 登出。
- 頭像：圓形、固定尺寸（如 28px）、與 Email 同一行、垂直置中。
- Email：過長截斷（如 20 字元 + …）、淺灰、單行。

## 四、實作要點（不變更內容）

1. **只改樣式與排版**  
   - 不新增、不刪除功能；不改文案（除已約定隱藏「取得 API 金鑰」）。
2. **頂部**  
   - 大標題 + 「‹ 儀表板」一列，樣式對齊參考圖。
3. **主導航**  
   - 維持 `st.sidebar.radio` 四項，用 CSS 做選中態（灰底圓角、字色變白）。
4. **底部**  
   - 用 spacer 或 flex 讓底部區視覺上固定在下；順序：設定 → 下雪 → 頭像+Email → 登出；頭像橘底白字、Email 截斷。
5. **CSS 範圍**  
   - 所有上述樣式僅作用於 `[data-testid="stSidebar"]`，不影響主內容區。
