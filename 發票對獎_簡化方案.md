# 發票對獎功能簡化方案

## 一、現狀問題

- 步驟多：選期別 → 自動取得 → 再按對獎；或貼上文字 → 解析 → 再按對獎。
- 資訊過多：開獎期別、特別獎／特獎／頭獎號碼、領獎期間、對獎結果摘要、明細表，全攤開。
- 選項多：12 期下拉 + 手動貼上 + 兩顆按鈕，初次使用容易困惑。

## 二、簡化原則

- **一步到位**：選「最新一期」或「上一期」後，**一鍵完成「取得開獎 + 對獎」**，不再分兩步。
- **結果優先**：頁面以「對獎結果」為主：一句摘要（中獎張數、總金額、領獎期限）+ 中獎明細表；開獎號碼改為**收合**，需要再展開。
- **手動收合**：手動貼上／解析改為 **expander「手動輸入開獎號碼」**，預設收合。

## 三、簡化後 UI 結構

```
🎰 發票對獎
  一句說明：選擇期別後一鍵對獎。

  [ 對獎（最新一期） ]  [ 對獎（上一期） ]
  → 點擊後：自動取得該期開獎號碼並對所有發票對獎，直接顯示結果。

  --- 對獎結果（有跑過對獎才顯示）---
  本期（114年11-12月）對獎 27 張，中獎 2 張，共 400 元。領獎期限至 115年5月5日止。
  [ 中獎明細表：日期 | 發票號碼 | 賣方 | 獎別 | 獎金 ]
  或：本期對獎 27 張，未中獎。

  ▶ 查看本期開獎號碼（收合）
     特別獎：xxx 特獎：xxx 頭獎：xxx, xxx, xxx  領獎期間自…

  ▶ 手動輸入開獎號碼（收合）
     貼上財政部頁面文字 + [ 解析並對獎 ]
```

## 四、具體調整

| 項目 | 現狀 | 簡化後 |
|------|------|--------|
| 主操作 | 選期別 → 自動取得 → 再按對獎 | **兩個按鈕**：「對獎（最新一期）」「對獎（上一期）」→ 取得+對獎一次完成 |
| 開獎號碼 | 取得後直接顯示特別獎/特獎/頭獎 | 收合在 expander「查看本期開獎號碼」 |
| 手動方式 | 文字框 + 解析鈕常駐 | 收合在 expander「手動輸入開獎號碼」 |
| 對獎結果 | 摘要 + 明細 + 重複標題 | **一句摘要** + **一張中獎明細表**（無則顯示「未中獎」） |
| 標題/說明 | 實驗版、多行 caption | 標題「🎰 發票對獎」+ 一句說明 |

## 五、小結

- 主流程：**兩個按鈕選期別 → 一鍵取得並對獎 → 只看一句摘要 + 明細表**。
- 開獎號碼與手動輸入改為**收合**，需要時再展開。
- 文案精簡，減少重複與選項，以「能識別中獎號碼」為前提，突出對獎結果。
